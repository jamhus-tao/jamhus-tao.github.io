简介

B 树 是多叉搜索平衡树。B 树 的结构更好的满足局部性原理，越矮的树的索引次数越少，从读写上提高代码效率。由于在磁盘访问中，数据读写占据了多数时间，因此 B 树 这种数据结构在数据库系统中有较好的效率。B 树 实际上很少使用，但它是 B+ 树、红黑树 的基础。

核心思想

​定义： m (m≥3)m~(m \ge 3)m (m≥3) 阶 B 树 的定义为：

  • 满足中序遍历升序。
  • 每个节点最多容纳 m−1m - 1m−1 个键。
  • 每个节点最少容纳 ⌈m2⌉−1\lceil\frac{m}{2}\rceil - 1⌈2m​⌉−1 个键，除根节点最少包含 111 个键。
  • 对于所有内部节点，如果节点容纳 kkk 个键，则其有 k+1k + 1k+1 个子节点。
  • 对于所有叶子节点，没有任何子节点。
  • 所有叶子节点必须位于同一层。

​时间复杂度：

由于 B 树 保证每个节点至少半满，且所有叶子节点位于同一层，因此树的最大深度是对数的。

中序遍历

插入元素

插入元素需要分 222 种情况讨论：

  • 插入后满足约束条件，无需继续维护。
  • 插入后超过约束条件，将节点以任意中位键分裂，中位键插入父节点，两分裂节点分别作中位键左右子节点。
  • 补充情况二，如果当前节点没有父节点，创建父节点。
  • 补充情况二，插入后父节点可能不满足，需继续向上维护。

如此过程中一定满足约束条件。

删除元素

删除元素需要分 444 类讨论：

  • 如果待删除键不是叶子节点，与其后继互换后删除后继。根据 B 树 定义，该情况后继一定存在且为叶子节点。(前驱也可)
  • 删除后满足约束条件，无需继续向上维护。
  • 删除后不足约束条件，分 222 类讨论：
    • 如果左或右兄弟任一删除一个键后依然满足约束，当前节点借用父键，父键借用一个兄弟节点键。操作后一定满足约束条件。
    • 如不满足上一条件，将当前节点、任意相邻兄弟节点、两者公共父键向下合并。此时父节点可能不满足约束，需继续向上维护。
    • 补充上一情况，如果父节点为根节点，删除后可能为空。此时需要删除父节点，令其唯一子节点为根。

如此操作过程中一定满足约束条件。
